### Fixed Table

Fixed tables allow for evenly spaces columns of data. Leverage this style when the information
being present to the user is all relatively the same length. Truncating cell data is generally 
frowned upon as is causes confusion for the user. There are rare exceptions but generally wrap text
to the next line.

<div class="disclaimer">
    If you have additional questions speak with a member of the Availity user experience team.
</div>

```jsx live=true viewCode=false
<div className="w-100 d-flex flex-row justify-content-around align-items-center">
  <table class="table table-fixed">
    <thead>
    <tr>
      <th class="col-md-1">#</th>
      <th class="col-md-3">Haiku</th>
      <th class="col-md-2">Last Name</th>
      <th class="col-md-5">Username</th>
    </tr>
    </thead>
    <tbody>
    <tr>
      <td>1</td>
      <td class="text-truncate">Haikus are easy; but sometimes they don't make sense; refrigerator</td>
```

### Hover Table

Hover tables are generally used in selection cases where then entire row can be clicked on. Hover
tables are great in any circustance where lots of information is being returned, they really help the 
readability of data dense user interfaces.

### Striped Table

Striped tables are not used very often anymore but are kept as part of the uikit offering. While they
can aid in the readabilkity of a table for the user we've found through testing users find the stiping
undesireable. We would recommnd going with a hover table in most cases. See above.
